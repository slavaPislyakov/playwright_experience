### Логика построения архитектуры фреймворка
    - Верхняя иерархия тесты
    - Из тестов вызывается степы страницы, которая наследуется от common степов:
      - Если компонента есть на ВСЕХ страницах: прописываем степы компоненты в коммон степах
      - Если компонента есть на 2 и более страницах, но НЕ НА ВСЕХ: прописываем степы компоненты точечно в нужно страницах
      - Если компонента только на этой странице: то вызываем PO для данной страницы
    - В PO компоненты описывается селекторы и простые методы типа: получения элемента/текста, дожидание элемента, и тд
